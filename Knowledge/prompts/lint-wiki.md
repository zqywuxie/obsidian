# 健康检查 (Lint)

> 定期对 Wiki 进行全面体检，发现并修复问题。

## 使用场景

当知识库已运行一段时间、积累了一定量的页面后，用此 prompt 做维护。

## Prompt

```
请对当前 Wiki 进行一次完整的健康检查（Lint）。

## 检查清单

### 1. 页面一致性
- [ ] 所有页面是否包含完整的 YAML frontmatter（title, tags, created, updated, sources）
- [ ] tag 命名是否一致（有无同义不同名的情况）
- [ ] 日期格式是否统一（YYYY-MM-DD）

### 2. 交叉引用
- [ ] 是否存在页面间提及了概念但未建立 [[Wikilink]] 的情况
- [ ] wiki/index.md 中的链接是否都能正确跳转
- [ ] 来源摘要页中的 "相关页面" 是否完整

### 3. 矛盾检测
- [ ] 不同来源对同一概念的说法是否有矛盾
- [ ] 如有矛盾：创建新页面或在已有页面中标注矛盾，说明各来源的证据

### 4. 过时内容
- [ ] 是否有早期来源的声明已被新来源修正或推翻
- [ ] 如有：更新相关页面，标注新旧来源的演变

### 5. 孤立页面
- [ ] 检查哪些页面没有入链（没有其他页面链接到它）
- [ ] 孤立页面是否需要保留？如需要，在其他页面添加引用链接

### 6. 知识缺口
- [ ] 哪些重要概念被提及但尚未创建独立页面
- [ ] 哪些问题可通过一次网络搜索填补数据空白

### 7. 索引完整性
- [ ] wiki/index.md 是否包含所有现有页面
- [ ] 统计信息是否准确

## 输出格式
对每个检查项，输出：
- ✅ 正常
- ⚠️ 需注意（附说明）
- ❌ 需修复（附具体修复方案）

## 修复步骤
健康检查完成后，逐项修复发现的问题，并在 wiki/log.md 中记录。
```
